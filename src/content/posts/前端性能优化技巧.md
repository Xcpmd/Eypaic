# 前端性能优化技巧

前端性能优化是提升用户体验的关键因素之一。本文将分享一些实用的前端性能优化技巧。

## 资源优化

### 图片优化

1. **使用适当的图片格式**：
   - JPEG适用于照片
   - PNG适用于透明背景的图片
   - WebP格式提供更好的压缩率

2. **响应式图片**：
   ```html
   <img src="image-small.jpg"
        srcset="image-medium.jpg 1000w, image-large.jpg 2000w"
        sizes="(max-width: 600px) 100vw, 50vw"
        alt="描述">
   ```

3. **懒加载图片**：
   ```html
   <img src="placeholder.jpg" data-src="real-image.jpg" loading="lazy" alt="描述">
   ```

### CSS和JavaScript优化

1. **压缩CSS和JavaScript**：使用工具如Webpack、Vite等压缩资源
2. **移除未使用的CSS**：使用PurgeCSS等工具移除未使用的样式
3. **代码分割**：将代码分割成多个块，按需加载

## 渲染优化

### 减少重排和重绘

1. **批量修改DOM**：使用DocumentFragment或一次性修改样式
2. **使用transform和opacity**：这些属性不会触发重排
3. **避免强制同步布局**：读取布局属性后再修改样式会导致性能问题

### 使用虚拟滚动

对于长列表，使用虚拟滚动技术只渲染可见区域的元素：

```javascript
import { FixedSizeList as List } from 'react-window';

const Row = ({ index, style }) => (
  <div style={style}>
    行 {index}
  </div>
);

const Example = () => (
  <List
    height={500}
    itemCount={1000}
    itemSize={35}
    width={300}
  >
    {Row}
  </List>
);
```

## 网络优化

1. **使用CDN**：将静态资源部署到CDN
2. **启用HTTP/2**：提高资源加载效率
3. **使用Service Worker**：实现离线缓存和资源预加载

## 监控和分析

使用工具如Lighthouse、WebPageTest等定期分析网站性能，找出性能瓶颈。

## 总结

前端性能优化是一个持续的过程，需要结合实际项目情况选择合适的优化策略。记住，过早优化是万恶之源，但忽视性能问题会影响用户体验。
