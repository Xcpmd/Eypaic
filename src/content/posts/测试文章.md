# Astro 博客模板

这是一个基于 Astro 构建的二次元风格现代化博客网站模板，专为技术宅的内容创作者和开发者设计。该模板提供了清晰的架构、优雅的视觉设计和良好的用户体验，让用户能够专注于创作高质量的内容。

## 网站架构

本博客网站采用静态站点生成器 Astro 构建，具有以下核心特性：
- 使用 Astro 的 Islands 架构，实现部分水合，提升性能
- 支持 Markdown 和 MDX 格式撰写文章
- 集成 RSS 订阅功能
- 响应式设计，适配各种设备屏幕
- 支持深色模式切换
- SEO 友好，包含 Open Graph 和 Twitter Card 元数据
- 支持代码高亮显示

## 页面布局

网站主要包含以下几个页面布局：

### 首页 (Index Page)
- 顶部导航栏，包含网站 Logo 和主要菜单项, 搜索框
- 图片背景, 中间为博客主要页面(以下代称为main-frame, 且是以下内容的父对象), 占据水平居中的60%, 两侧为露出的背景
- 英雄区域(水平方向占满main-frame, 垂直方向30%)
  - 左侧为头像(带边框), 
  - 右侧上部为博客主人名字, 下部为个人简介及个人tag(个人成分,比如python、日漫等, 可自定义)
- 最新文章列表(可置顶, 可选带封面), 每篇文章显示标题、摘要、发布日期和标签
- main-frame 的右侧有竖直排列的长条形离岛型悬浮框, 可以选择将文章按照tag/时间轴/文章所属集合/默认顺序排列.
- 底部(footer)包含版权信息和各种仓库的跳转链接(github/gitee, 链接暂时指向为对应网站的主页, 具体链接以后补充)

### 文章详情页 (Post Page)
- 文章标题、文章tag、发布时间和阅读时长/字数,
- 文章正文内容，支持 Markdown 格式
- 相关文章推荐
- 上下篇跳转
   - 默认按照文章的所属集合内的发布顺序排列, 否则按总时间轴排列
- 评论区（可集成第三方评论系统）

### 归档页 (Archive Page)
- 按时间轴排列的所有文章列表
- 支持按年份或月份筛选

### 分类页 (Category Page)
- 显示特定分类下的所有文章

### 标签页 (Tag Page)
- 显示特定标签下的所有文章

### 集合页 (Collection Page)
- 将文章按照集合分类
  - 进入后仅显示各个集合, 点击集合后打开集合显示内部文章

### 关于页 (About Page)
- 个人或团队介绍
- 联系方式
- 个人项目/开源仓库链接
- 项目用到的技术栈(比如python、django、vue等)

## 视觉风格

网站采用现代简约的设计风格, 以无色毛玻璃为主, 且带有适当阴影，注重可读性和用户体验：

- 默认主色调：天蓝色(部分强调色)
  - 在导航栏有二级菜单的滑块可调节(包含所有色彩类型)
- 字体：目前先使用系统默认字体栈, 后续补充(预留字体文件夹, 在其中放入字体文件后使用)
- 排版：合理的行高和间距(配置文件可调)，提升阅读体验
- 响应式设计：适配手机、平板和桌面设备
- 深色模式：提供深色主题/跟随系统选项，减少夜间阅读眼部疲劳, 切换时存在动画:
    - 由亮至暗: 从页面外侧出现一个以切换按钮为圆心的快速缩小的圆, 其扫过的地方变为暗色主题
    - 由暗至亮: 与变暗的动画相反
- 动画效果：舒适的过渡动画, 弹窗动画、二级菜单、页面跳转等动画要完整，增强交互体验但不干扰阅读

## 用户浏览体验

当用户访问网站时，他们将经历以下流程：

1. **首页加载**：懒加载, 用户首先看到的是一个全屏的毛玻璃效果将页面覆盖(此时页面在加载, 效果持续至加载结束, 加载不包括背景), 并在页面上显示欢迎语, 并且带有手写动画(我已有现成代码, 不需要管, 为我在components/预留一个空的hello.astro文件并将其作为组件插入到此全屏的中心, 我将会在文件里面添加我的代码, 带动画), 加载完成后此界面的毛玻璃效果缓慢过渡地减弱并消失(消失后直接删除元素)
2. **导航浏览**：通过顶部导航栏，用户可以访问不同页面，如归档、分类、标签和关于页面, 搜索框, 主题色调整的二级菜单触发按钮, 全部都是毛玻璃风格(毛玻璃强度不同以凸显层次感)。
3. **文章阅读**：点击感兴趣的文章标题后，用户进入文章详情页，可以完整阅读文章内容。
4. **互动交流**：在文章底部，用户可以查看相关文章推荐，并参与评论讨论。
5. **搜索探索**：通过标签云或分类列表，用户可以发现更多相关内容。
6. **文章阅读**: 在左侧显示文章的大纲轴并且可点击跳转至对应位置

## 补充功能

此博客模板还具备以下附加功能：

- 文章搜索功能
- 社交媒体分享按钮
- 文章阅读进度指示器
- 返回顶部按钮
- 多语言支持(目前仅支持英文和中文, 后续根据需求添加)
- src目录下存在一个名为config.ts的配置文件, 设置网站内容, 包括博主的名字, 简介, 默认主题色等(可参考/public/config.ts)
- 文章的markdown文件需要在/content/posts/目录下, 文件名即为文章的slug, 文章的frontmatter中需要包含title, date, tags, categories, description等信息(可参考public/Muzuki/content/posts/first-post.md), 在博客中则根据这些信息显示文章的标题, 字数, 发布时间, 标签, 分类, 描述等,并自动生成文章的大纲轴和预期阅读时间

## 配置文件

- 配置文件为/src/config.ts, 可根据需要修改, 包括博主的名字, 简介, 默认主题色, 文章的目录, 文章的frontmatter中的字段等
- 包括博主的个人tags.
- 博客背景的显示方式:
  - 如果assets/*/下存在不少于一张图片, 这可以配置为随机显示其中一张图片作为博客的背景或者所有图片轮播显示
  - 如果只存在一张图片, 则默认使用这张图片作为博客的背景
  - 可以在配置文件中设置是否打乱轮播顺序, 以及轮播的时间间隔等

## 🧞 命令说明

所有命令都从项目根目录运行：

| 命令                    | 操作                              |
| :--------------------- | :--------------------------------- |
| `pnpm install`         | 安装依赖                           |
| `pnpm dev`             | 启动本地开发服务器 `localhost:4321` |
| `pnpm build`           | 构建生产站点到 `./dist/`            |
| `pnpm preview`         | 在部署前本地预览构建结果             |
| `pnpm astro ...`       | 运行 CLI 命令如 `astro add`, `astro check` |
| `pnpm astro -- --help` | 获取 Astro CLI 帮助信息              |
| `pnpm add-post`        | 添加新文章(会在/content/posts/目录下创建一个新md文件, 文件名即为文章的slug, 文章的frontmatter中默认包含title, date, tags, categories, description等信息) |
